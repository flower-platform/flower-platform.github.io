The special first scenario: _quickInstructions()

The principle "Tests Are Demo" == "also use tests as demo" has a drawback in practice:

  • The scenarios contain too much details. This is good for the tests (because richer test => bigger coverage).
  • But bad for a quick read focused on understanding the big picture, while leaving out the details.

The antidote 💉 is the following convention: have a special first scenario (_quickInstructions()):

  • w/o code;
  • only w/ comment = some quick instructions 📄.

❗IMPORTANT: Quick instructions 📄, and not instructions 📄📄📄. Some hints:

  • Use picture(s); annotated w/ comments below.
  • It's OK to omit details. Favor brevity over exhaustiveness. People will probably discover them anyway. And/or they will look at the scenarios, where there is plenty of details.
  • Avoid multiple pictures of the same screen. Similar pictures (w/ relatively small differences) tend to de-focus people.

Use empathy. Put yourself in the reader's shoes 👞, 👟, 👢.

  • He/she is browsing the doc in order to discover the N (= 27) features of your app. If 📄 x 27 => 🙂. If 📄📄📄📄📄📄📄📄 x 27 => 😴.
  • Or better. Think a fancy microwave oven. It has a manual of 180 pages. But also a one pager, w/ about 70% of the notions 🙂.
  • Or even better. Think a vacuum cleaner. It has a manual of 97 pages. Probably a picture + about 10 words contain 95% of the features 🙂.

Example 1​

  1. A row = one Todo. Priority -> color of the top border. You can use the checkbox to toggle "done/not done".
  2. Some buttons for one row/Todo. ⬆️/⬇️ move the Todo up/down.
  3. Totals. Serves also as legend for priority <-> color.

tip
  • I avoid to say "Pressing the Add button ..."
  • Actually I don't say anything about the buttons that have labels. I do mention them in the image (w/ arrow or red numbers), but that's it. E.g. what should I say about the "Delete" button?
  • For the second image I don't say actually anything. It's important to have the image, but in this case I think no words are needed.

Example 2: better​

  1. Shows the current organization. To change: click, ...
  2. ... and then click in the tree.
  3. By default, only data from the current org is shown (e.g. in various pages/tables).
  4. You can change this; i.e. see data from current org AND data from all your orgs.
  5. Or data from the current org AND data from some orgs (use the checkboxes in the tree to select).

In the tree: 🔵blue = current org; 🟡yellow = checked = an additional org that is also included, cf. 4/ or 5/

  1. Click to change the current org. BTW, this is the same tree as the one above
  2. Shortcut for 4/
  3. Shortcut; like clicking cf. 1/

Example 2: not so good​

tip

Both the previous paragraph/example and this talk about the same feature. The previous example is newer/better. It fits better into the category "quick instructions". This one (an initial experiment) is rather "instructions", and not "quick instructions". We keep both to illustrate the difference.

Note:​

Organization tree from the left side is equivalent with the one from the top right organization dropdown.

The items represents the associated organizations taken from the user roles. A user with a role with no organization will have access to all organizations.

1. By clicking on organization label from the top right, the advanced organization dropdown will be opened. Also by clicking on More... button from the left side. Users with access to all organizations will have All organizations as the first tree item.

2. By checking All my organizations grayed/checked checkboxes will be displayed on tree items and will be colored in orange (only selected one will remains blue). All my orgs from the left side have the same effect. The option is not displayed if the user have access to all organizations.

3. By checking Checked organizations editable checkboxes will be displayed on tree items. Checked items are also colored in orange.

4. By checking Nothing no checkboxes will be displayed on tree items.

5. By clicking on a tree item it will close the popup and will change the current organization with the one selected.

6. By clicking on Apply button the popup will be closed and the selection regarding other organizations will be applied.

7. After applying the changes, the table contains 5 records, including additional records for organizations company1.munich and company1.paris, besides the records with the organization company2.london which is the current organization selected.
