VS Code extension

To be installed from here.

It is published also in the Open VSX Registry so that it can be used in the online instance of VS Code embedded in GitLab. Extensions are not enabled by default there. Your GitLab admin needs to enable them. Then each user needs to activate them as well (Click on user icon > Preferences > scroll to Integrations > check Enable extension marketplace).

tip

We designed Featurebook to leverage the "paste image" feature of VS Code. It works like this:

  • capture a screenshot (i.e. the image will be stored in the clipboard);
  • select a dir, and paste (CTRL + V) => a new file will be created there, w/ the image from clipboard.

To use this combined w/ Featurebook:

1
2
Click, for the scenario you want to add images.

3
A corresponding dir will be created and/or focused it in the tree.

4
Paste (CTRL + V) the image (that you have in the clipboard; e.g. a screenshot you just made). Tip:

  • if you have only one image in your scenario, leave the name "as is" (i.e. image.png). It won't collide w/ other scenarios.
  • if you have multiple images, then you should rename each one w/ a meaningful name.

5
Reference the image using @img from the scenario. You can access images from other scenarios using e.g. ../otherScenario/image.png.

6
And there you go: the preview is updated w/ the image.

7
BTW: this is the place where the .md file is generated. You can open it from the web (e.g. GitHub); the images will display.
