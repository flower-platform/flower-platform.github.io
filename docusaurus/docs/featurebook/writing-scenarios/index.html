Writing scenarios

/**
 * * THEN1 `Todo.priority` -> color of the top border
 * * THEN2 `Todo.text`
 * * THEN3 `Todo.done`: when `true`, we 'line through' the text
 * * THEN4 `Todo.done`: when `true` we **CHECK** the checkbox
 * * THEN5 idem THEN1
...
 * * THEN12 Number of Todos w/ priority = High
 */
@Scenario()
async whenScreenTodosShows() {
...
  • The name of the function always starts w/ when...(), i.e. the (first) trigger of the test. Hence there is no WHEN ... section in the comment.
    • For UI oriented trigger actions use the e.g. whenScreenAbcShows(), whenButtonOkClicked(). I.e. pattern whenSomeComponentSomeAction().
    • For more unit tests style, e.g. whenCalculate() for testing the function calculate().
  • In the comment, use bullets and the keywords GIVENx, WHENx, THENx. The "x" is needed to provide uniqueness to the bullets.

Each bullet will need to be explicitly referenced from the code of the test. No reference => runtime error. E.g.:

tad.ref("THEN12");
await tad.assertWaitable.equal(tad.screenCapturing.getByTestId(testids.totalHigh).textContent, "2")

Dependent tests​

...
@Scenario()
async andWhenButtonAddClicked() {

Use andWhen...(). The "and" signifies that the test/function is dependent on what happened in the previous.

Multiple WHENs​

/**
 * * THEN the form opens
 * * THEN1 first text field is already focused. So that people can directly start typing, w/o needing to click here first
 * ---
 * * WHEN2 type and save
 * * THEN2 The newly added Todo appears at the bottom of the list
 */
@Scenario()
async andWhenButtonAddClicked() {

A test w/ multiple trigger actions, not just the initial one (cf. function name).

  • Separate w/ ---. Separation w/ empty line is no go; the rendered Markdown is not nice.
  • Use WHEN2, WHEN3, etc. Don't use WHEN1; it is reserved for the trigger from the function name.

Series​

/**
 * Normal use.  
 * 
 * * THEN ...
 */
whenButtonShowReportClick01() {
...

/**
 * No data.
 * 
 * * THEN ...
 */
whenButtonShowReportClick02() {
...

/**
 * Invalid period.
 * 
 * * THEN ...
 */
whenButtonShowReportClick03() {
...

Use numbered suffixes (...1(), ...2()) + a brief explanation in the comment. We tried in the past to put the explanation in the name of the function. It was ugly.
