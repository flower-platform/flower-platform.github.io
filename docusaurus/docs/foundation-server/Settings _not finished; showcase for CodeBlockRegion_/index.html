Settings _not finished; showcase for CodeBlockRegion_

info

This page is to demo/showcase the rough structure of a feature documentation.

It's not (yet) the full doc for Settings. We'll update when its time will come.

For devs​

1
We create a class with the settings that we need. This will be editable from the UI.

Snippet extracted from EmployeeService.java (region settings1-class, between lines [34, 40])
@JsonTypeName("EmployeeServiceHeaderSettings")
public static class EmployeeServiceHeaderSettings extends AbstractSettings {
	public Boolean showBasicSettingsButton;
	public Color settingsButtonColor;
	public String postItMessage;
	public String logo;
}

2
In order to access it, we use a bean of type SettingsCache. So we define the bean in one of the classes annotated w/ @Configuration. With this occasion, we also define the default value:

Snippet extracted from LeaveManagementApp.java (region settings1-bean, between lines [95, 102])
@Bean
public SettingsCache<EmployeeServiceHeaderSettings> employeeServiceHeaderSettings() {
	return new SettingsCache<>(ServerUtils.jsonLike(new EmployeeServiceHeaderSettings(), s -> {
		s.showBasicSettingsButton = false;
		s.settingsButtonColor = Color.blue;
		s.postItMessage = "";
	}));
}

3
Inject this bean there where you need it:

Snippet extracted from EmployeeService.java (region settings1-field, between lines [62, 63])
@Inject
protected SettingsCache<EmployeeServiceHeaderSettings> employeeServiceHeaderSettings;

4
And use it:

Snippet extracted from SettingsEntityServiceTest.java (region settings1-temp-usage, between lines [258, 261])
// for global
ReflectionAssert.assertLenientEquals(expectedForGlobal, employeeService.getEmployeeServiceHeaderSettings().get(null));
// for company1
ReflectionAssert.assertLenientEquals(expectedForCompany1, employeeService.getEmployeeServiceHeaderSettings().get("company1"));
